# 农业正式调研示例

## Prompt

```text
使用 数据集标准调研 对农业高质量数据集建设相关文件做正式调研。

调研目标：
1. 系统梳理农业高质量数据集建设相关的官方文件，既包括国内通用顶层政策和标准，也包括农业专项政策、农业行业标准、官方典型案例和必要的国际参考。
2. 不限制只找 5 个或 10 个文件。先形成尽可能完整的候选清单，再从完整清单中推荐 5-8 个最值得优先阅读的文件。

文件范围：
1. 顶层设计与政策框架：数字中国、数据基础制度、数据基础设施、国家数据标准体系、数据要素、人工智能、行业高质量数据集建设等。
2. 高质量数据集专项文件：建设指引、建设指南、格式要求、分类指南、质量评测规范、数据标注要求等。
3. 支撑性通用标准：数据质量、数据治理、数据管理成熟度、数据标注、数据安全、数据分类分级、生成式人工智能训练数据安全等。
4. 农业专项文件：智慧农业、农业大数据、农业数据共享、农业数据交换、农业核心元数据、农业大数据安全、农业地理信息数据管理等。
5. 官方典型案例：国家数据局、农业农村部或省级数据主管部门发布的农业高质量数据集案例。
6. 如补充国际文件，请单独列为“国际参考”，不得与国内正式标准混在一起。

输出要求：
1. 严格使用以下固定表头：
   文档名称｜发布机构和时间｜文档类型｜适用行业或场景｜与高质量数据集构建的关系｜核心内容摘要｜可参考的质量指标｜官方链接
2. 官方链接必须是具体通知页、详情页、标准详情页或案例页面，不使用门户首页、搜索首页、第三方转载页或直接 PDF 链接替代。
3. 标准类文件必须优先使用标准主管平台的具体标准页，例如 `openstd.samr.gov.cn/bzgk/std/newGbInfo?hcno=...`、`std.samr.gov.cn/gb/search/gbDetailed?id=...`、`hbba.sacinfo.org.cn/stdDetail/...`。标准销售、交易、购买、付费在线阅读等商业服务网站一律排除，不纳入检索范围。
4. 每一行都要核验文档名称、标准号或文号、发布机构、发布时间、实施状态和官方来源。
5. 对“可参考的质量指标”要区分：标准明确指标、文件原文要求、根据文件内容提炼出的参考指标。不得把推导指标写成原文指标。
6. 如果有官方 PDF 或附件，请下载到输出目录的 pdfs/ 文件夹；如果不能下载或正文受版权限制，请在下载记录中说明原因。

交付文件：
1. research_table.md：主表格。
2. research_table.xlsx：如环境支持，生成 Excel 版本。
3. download_log.csv：记录来源网页、附件链接、本地文件名、下载状态和备注。
4. summary.md：总体分析总结。
5. source_notes.md：记录未获取全文、征求意见稿、标准计划、非官方来源排除等说明。

总结要求：
1. 总结高质量数据集构建通常包括哪些关键环节。
2. 总结农业领域相较医疗、工业、交通等行业在数据采集、时空对齐、多模态融合、标注、质量控制和合规方面的特点。
3. 从完整清单中推荐 5-8 个重点阅读文件，并说明推荐理由。
4. 明确哪些文件是正式实施文件，哪些是征求意见稿、技术文件、典型案例或标准计划。
```

## 期望结果

- 不是只输出 5-10 个文件，而是先形成完整清单，再给重点阅读推荐。
- 至少覆盖五类文件：顶层设计与政策框架、高质量数据集专项文件、支撑性通用标准、农业专项文件、官方典型案例。
- 表格链接为具体官方网页，不使用门户首页或搜索首页；标准销售、交易、购买、付费在线阅读网站不进入候选来源。
- 对无法获取全文、无法下载 PDF、标准正文受版权限制等情况有单独说明。
- 如果执行下载，应生成 `download_log.csv`，而不是只在正文里简单说“已下载”。

## 检查重点

- 是否严格使用固定表头。
- 是否包含顶层政策，而不是只收集农业行业标准。
- 是否区分标准明确指标与可参考指标。
- 是否给出 5-8 个重点阅读文件，而不是一开始就只找 5-8 个文件。
- 是否说明征求意见稿、技术文件、典型案例和正式标准的状态差异。
