# 通用领域正式调研模板

这个模板适用于农业、医疗、工业制造、交通、金融、能源、教育、科研数据等领域。

## 怎么替换

使用时只需要改两类位置：

1. 把全文所有 `【目标领域】` 替换成你的行业名称，例如 `医疗`、`工业制造`、`交通`、`能源`。
2. 把 `文件范围` 第 4 条中的行业专项文件说明，替换成该行业自己的政策、标准、数据规范和典型场景。

最省事的做法：

1. 先复制下面完整 Prompt。
2. 用查找替换，把 `【目标领域】` 全部替换成目标行业。
3. 再手动修改 `文件范围` 第 4 条。
4. 其他部分一般不用改。

示例：如果目标是医疗，先把所有 `【目标领域】` 替换成 `医疗`，再把第 4 条改成：

```text
4. 医疗专项文件：健康医疗大数据、全民健康信息化、电子病历、医学影像 AI、医疗器械 AI、个人信息保护、伦理审查、临床数据质控、医疗数据安全和监管要求等。
```

## Prompt

```text
使用 数据集标准调研 对【目标领域】高质量数据集建设相关文件做正式调研。

调研目标：
1. 系统梳理【目标领域】高质量数据集建设相关的官方文件，既包括国内通用顶层政策和标准，也包括【目标领域】专项政策、行业标准、官方典型案例和必要的国际参考。
2. 不限制只找 5 个或 10 个文件。先形成尽可能完整的候选清单，再从完整清单中推荐 5-8 个最值得优先阅读的文件。

文件范围：
1. 顶层设计与政策框架：数字中国、数据基础制度、数据基础设施、国家数据标准体系、数据要素、人工智能、行业高质量数据集建设等。
2. 高质量数据集专项文件：建设指引、建设指南、格式要求、分类指南、质量评测规范、数据标注要求等。
3. 支撑性通用标准：数据质量、数据治理、数据管理成熟度、数据标注、数据安全、数据分类分级、生成式人工智能训练数据安全等。
4. 【目标领域】专项文件：该领域主管部门政策、行业标准、数据安全规范、元数据规范、数据交换规范、质量控制规范、监管要求和技术指南等。
5. 官方典型案例：国家级、部委级、省级主管部门或权威行业机构发布的【目标领域】高质量数据集案例。
6. 如补充国际文件，请单独列为“国际参考”，不得与国内正式标准混在一起。

输出要求：
1. 严格使用以下固定表头：
   文档名称｜发布机构和时间｜文档类型｜适用行业或场景｜与高质量数据集构建的关系｜核心内容摘要｜可参考的质量指标｜官方链接
2. 官方链接必须是具体通知页、详情页、标准详情页或案例页面，不使用门户首页、搜索首页、第三方转载页或直接 PDF 链接替代。
3. 标准类文件必须优先使用标准主管平台的具体标准页，例如 `openstd.samr.gov.cn/bzgk/std/newGbInfo?hcno=...`、`std.samr.gov.cn/gb/search/gbDetailed?id=...`、`hbba.sacinfo.org.cn/stdDetail/...`，不要把交易、购买或在线服务页作为主表官方链接。
4. 每一行都要核验文档名称、标准号或文号、发布机构、发布时间、实施状态和官方来源。
5. 对“可参考的质量指标”要区分：标准明确指标、文件原文要求、根据文件内容提炼出的参考指标。不得把推导指标写成原文指标。
6. 如果有官方 PDF 或附件，请下载到输出目录的 pdfs/ 文件夹；如果不能下载或正文受版权限制，请在下载记录中说明原因。

交付文件：
1. research_table.md：主表格。
2. research_table.xlsx：如环境支持，生成 Excel 版本。
3. download_log.csv：记录来源网页、附件链接、本地文件名、下载状态和备注。
4. summary.md：总体分析总结。
5. source_notes.md：记录未获取全文、征求意见稿、标准计划、非官方来源排除等说明。

总结要求：
1. 总结高质量数据集构建通常包括哪些关键环节。
2. 总结【目标领域】相较其他行业在数据采集、多源融合、标注、质量控制、合规和应用验证方面的特点。
3. 从完整清单中推荐 5-8 个重点阅读文件，并说明推荐理由。
4. 明确哪些文件是正式实施文件，哪些是征求意见稿、技术文件、典型案例或标准计划。
```

## 检查重点

- 是否把【目标领域】专项文件替换成了真实行业文件，而不是继续沿用农业示例。
- 是否包含顶层政策和高质量数据集专项文件，而不是只找行业案例。
- 是否先形成完整候选清单，再推荐重点阅读文件。
- 是否区分标准明确指标、文件原文要求和可参考指标。
- 是否对无法下载正文或标准全文受限的情况做了说明。
