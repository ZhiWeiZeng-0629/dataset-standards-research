# 测试清单

使用本 skill 输出结果后，按以下清单检查。

## 触发与结构

- [ ] 是否明显按“数据集标准调研”流程工作，而不是普通搜索摘要。
- [ ] 是否严格使用固定表头。
- [ ] 是否包含顶层设计与政策框架。
- [ ] 是否包含高质量数据集专项文件。
- [ ] 是否包含目标行业专项文件。
- [ ] 完整调研是否先形成完整清单，再推荐重点阅读文件。

## 检索与来源

- [ ] 国内文件是否中文优先检索。
- [ ] 是否优先使用官方来源。
- [ ] 官方链接是否为具体通知页、详情页或标准页。
- [ ] 是否避免门户首页、搜索首页、第三方 PDF、文库和网盘。
- [ ] 标准号、文号、发布时间和实施状态是否可核验。

## 指标与摘要

- [ ] 是否区分“标准明确指标”和“可参考质量指标”。
- [ ] 是否避免把推导指标写成原文指标。
- [ ] 是否对征求意见稿、标准计划、技术文件、典型案例进行状态区分。
- [ ] 是否说明无法获取全文或 PDF 的原因。

## 输出质量

- [ ] 是否没有记忆引用、工具痕迹或调试信息。
- [ ] 总结是否覆盖通用共性、顶层逻辑、行业特点、重点阅读和后续补充。
- [ ] 对行业差异的描述是否具体，不只是泛泛而谈。
